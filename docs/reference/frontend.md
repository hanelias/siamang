# `siamang.frontend` — Bundle Builder and Runtimes Reference

The `frontend` subpackage is responsible for compiling a `Questionnaire` into a deployable static bundle (consisting of HTML, JS, CSS, and environment configuration) [1]. While end-users typically interact with this pipeline indirectly via the high-level `survey.deploy(...)` method, this document provides the comprehensive API reference for the underlying compilation pipeline, theming engine, and client adapters [2].

```python
from siamang.frontend import (
    FrontendBuilder, UIConfig,
    SurveySchema, SurveyBundle,
    ReactRuntime, SurveyJSRuntime, RuntimeAdapter, RuntimeRenderContext,
    BackendClientTemplate, LocalClientTemplate, SupabaseClientTemplate, ClientEnv,
    compile_questionnaire, compile_expression, expression_variables,
    THEME_PRESETS, compile_css, get_preset,
)
```

---

## `SurveySchema` — The Intermediate Representation

The `SurveySchema` class is a frozen, immutable dataclass representing the platform-agnostic, intermediate representation of a survey. It is generated by calling `compile_questionnaire(...)` on a live `Questionnaire` instance [2].

```python
@dataclass(frozen=True, slots=True)
class SurveySchema:
    title: str
    pages: list[dict[str, Any]]
    variables: dict[str, dict[str, Any]]
    language: str = "en"
    description: str | None = None
    completion_text: str = "Thank you for your participation!"
    show_progress: bool = True
    allow_back: bool = True
    one_question_per_page: bool = False
    deadline: datetime | None = None
    max_responses: int | None = None
    quotas: list[dict[str, Any]] = []
    metadata: dict[str, Any] = {}
```

#### Methods

* `to_surveyjs() -> dict[str, Any]`: Renders the intermediate representation into a JSON structure compatible with the SurveyJS library [1].
* `to_dict() -> dict[str, Any]`: Serializes the entire schema into a JSON-compatible dictionary, including standard metadata like `format_version=1` and ISO-formatted deadlines.

## `SurveyBundle`

A `SurveyBundle` is an immutable container that holds all the physical files (HTML, CSS, JS, and config) that make up a compiled survey, ready to be written to a local disk or uploaded to a cloud hosting provider [2].

```python
@dataclass(frozen=True, slots=True)
class SurveyBundle:
    files: dict[str, str | bytes] = {}     # {relative_path: content}
    manifest: dict[str, Any] = {}          # {"format_version", "survey_id", "title", "runtime", ...}
```

#### Methods

| Method | Return Type | Description |
| :--- | :--- | :--- |
| `write_to(target: str \| Path)` | `Path` | Writes every file in the bundle to the specified target directory, creating parent directories if they do not exist. |
| `to_zip() -> bytes` | `bytes` | Generates a deflate-compressed ZIP archive containing all the files in the bundle. |
| `manifest_json() -> str` | `str` | Returns a pretty-printed JSON string representation of the bundle's manifest. |
| `compute_digest() -> str` | `str` | Computes a 16-character SHA-256 prefix hash over all file contents in the bundle, useful for versioning and caching. |
| `with_hashed_filenames()` | `SurveyBundle` | Returns a new bundle where all static assets (`.js` and `.css`) are renamed to include a content hash (e.g., `app.js` → `app.a1b2c3d4.js`) to prevent browser caching issues. HTML file references are updated automatically. |

## `FrontendBuilder`

The `FrontendBuilder` orchestrates the compilation pipeline. It combines a `RuntimeAdapter`, a `UIConfig` design system, and a `BackendClientTemplate` to assemble a complete, deployable `SurveyBundle` [2].

```python
@dataclass(frozen=True, slots=True)
class FrontendBuilder:
    runtime: RuntimeAdapter = SurveyJSRuntime()
    ui: UIConfig = UIConfig()

    def build(
        self,
        schema: SurveySchema,
        *,
        client: BackendClientTemplate,
        env: ClientEnv,
        survey: Questionnaire | None = None,
    ) -> SurveyBundle: ...
```

When building, the returned bundle contains:
* `index.html`: The main survey entry point.
* `closed.html`: The screen displayed when a survey has expired, reached its response limit, or when quotas are full.
* `style.css`: The compiled stylesheet containing the active theme.
* `env.js`: The runtime environment variables injected at deployment.
* `manifest.json`: Metadata about the active runtime, survey ID, and version.

> **Note on Runtimes**: If using the `ReactRuntime`, the live `Questionnaire` object must be passed to the `survey` parameter during build, as the React compiler compiles expressions directly from the Python source objects [2].

## `UIConfig` — The Design System

The `UIConfig` dataclass defines the visual appearance, typography, layout, branding, and localization strings for the deployed survey. Siamang defaults to a clean, highly readable "academic" design system featuring a narrow line measure, comfortable vertical spacing, and a classic serif body font [3].

```python
@dataclass(frozen=True, slots=True)
class UIConfig: ...
```

### 1. Color Palette

All colors are specified as standard CSS hex color codes (e.g., `"#2c5f8a"`).

| Property | Default | Description |
| :--- | :--- | :--- |
| `primary_color` | `"#2c5f8a"` | The dominant brand color, used for primary buttons, active states, and highlights. |
| `accent_color` | `None` | An optional accent color. If `None`, it automatically falls back to `primary_color`. |
| `background_color` | `"#fbfbfb"` | The page background color. |
| `surface_color` | `"#ffffff"` | The background color of cards, panels, and input containers. |
| `text_color` | `"#1a1a1a"` | The primary body text color. |
| `muted_text_color` | `"#5a5a5a"` | The color for secondary text, hints, and placeholder values. |
| `border_color` | `"#e6e4df"` | The color for dividers, input borders, and card outlines. |
| `error_color` | `"#b3261e"` | The color for validation errors and alert indicators. |
| `error_soft_color` | `"#fdf1f0"` | A light background tint used behind validation error messages. |
| `warn_color` | `"#9a6a1a"` | The color for warning states. |

### 2. Typography

Siamang features an integrated web-font loader. By default, it loads high-quality typography from the Google Fonts CDN [2].

| Property | Default | Description |
| :--- | :--- | :--- |
| `font_preset` | `"academic"` | The active typography configuration. Allowed values: `"academic"`, `"modern"`, or `"humanist"`. See the [Typography Presets](#typography-presets) section below. |
| `font_family` | *Source Serif 4 stack* | The primary body font family. |
| `heading_font_family` | `None` | The font family used for headers. If `None`, it mirrors `font_family`. |
| `ui_font_family` | *Inter stack* | The font family used for user interface controls, buttons, and form inputs. |
| `mono_font_family` | *JetBrains Mono stack* | The font family used for code blocks and preformatted text. |
| `font_size` | `"15.5px"` | The base body font size. |
| `line_height` | `"1.6"` | The body line spacing, optimized for reading comfort [3]. |
| `font_pair` | `"serif"` | Preconfigured heading-to-body font pairing style. Allowed values: `"serif"`, `"sans"`, or `"mixed"`. |

### 3. Layout and Density

| Property | Default | Description |
| :--- | :--- | :--- |
| `width` | `"700px"` | The maximum container width. Keeping this under `750px` ensures an optimal line measure of 60–75 characters per line, which is recommended for reading comprehension [3]. |
| `radius` | `"4px"` | The border-radius applied to cards, buttons, and form inputs. |
| `density` | `"comfortable"` | The overall vertical and horizontal spacing density. Allowed values: `"compact"`, `"comfortable"`, or `"spacious"`. |
| `question_style` | `"plain"` | The visual container style for questions. Allowed values: `"plain"` (minimal spacing), `"divided"` (separated by lines), `"carded"` (each question is rendered inside its own raised card), or `"accent"` (adds a vertical brand line next to the active question). |

### 4. Header and Branding

| Property | Default | Description |
| :--- | :--- | :--- |
| `logo_url` | `None` | Optional URL pointing to a logo image displayed in the header. |
| `logo_text` | `None` | Short text logo displayed when `logo_url` is not set. If omitted, it is automatically derived from the initials of `institution_name`. |
| `logo_position` | `"left"` | The alignment of the logo in the header. Allowed values: `"left"`, `"right"`, or `"center"`. |
| `show_title` | `True` | If `True`, displays the questionnaire's title in the header. |
| `institution_name` | `None` | The name of the academic institution or research organization, displayed beneath the title. |
| `study_subtitle` | `None` | Subtitle text displayed below the institution name. |
| `show_section_numbers` | `True` | If `True`, displays page numbers within the progress indicator. |
| `show_progress_text` | `True` | If `True`, displays text-based progress (e.g., `"Page 2 of 5"`) in the progress bar. |
| `estimated_minutes` | `None` | Optional estimate of the time required to complete the survey, displayed on the welcome page. |

### 5. Footer

| Property | Default | Description |
| :--- | :--- | :--- |
| `privacy_url` | `None` | Optional link to the organization's privacy policy, displayed in the footer. |
| `contact_email` | `None` | Optional contact email address, rendered as a `mailto:` link in the footer. |
| `ethics_statement` | `None` | Optional text block displaying the research ethics approval statement or reference number. |

### 6. Localisation UI Overrides

These properties allow researchers to translate or customize the text of default user interface buttons and dialogs. All properties default to English if unset.

```python
# Available localization override fields:
next_button_text, prev_button_text, submit_button_text, submitting_text,
required_text, saving_text, select_placeholder, of_text, selected_text,
resume_title, resume_action, restart_action, page_text, of_total_text,
retry_title, retry_body, retry_action, save_local_action,
completion_title, completion_body
```

### 7. Navigation, Access Control, and Analytics

| Property | Default | Description |
| :--- | :--- | :--- |
| `progress_style` | `"bar"` | The style of the progress indicator. Allowed values: `"bar"` (linear progress bar), `"dots"` (step-by-step dots), or `"both"`. |
| `default_theme` | `"light"` | The default color theme. Allowed values: `"light"`, `"dark"`, or `"system"` (respects browser preference). |
| `redirect_url` | `None` | Optional URL to automatically redirect the respondent to after successful submission (triggered after a 5-second countdown). |
| `allow_back` | `True` | If `True`, displays the "Previous" button. Setting this to `False` disables backward navigation, which is useful for fixed-order surveys where allowing respondents to modify past answers would break branching logic or experimental designs [4]. |
| `enable_analytics` | `False` | If `True` and deployed to Vercel, automatically injects the Vercel Web Analytics script. |
| `require_access_code` | `False` | If `True`, displays an access gate requiring respondents to enter a valid code before starting the survey. |
| `access_codes` | `None` | A list of valid access codes (e.g., `["user-001", "user-002"]`). |
| `access_title` | `None` | The title displayed on the access gate screen. |
| `access_body` | `None` | The body text displayed on the access gate screen. |
| `access_placeholder` | `None` | The placeholder text inside the access code input field. |
| `access_button` | `None` | The text of the submit button on the access gate screen. |

### 8. Custom Stylesheets

* `custom_css: str | None = None`: A raw CSS string appended to the compiled stylesheet. This serves as an escape hatch, allowing researchers to override any aspect of the design system.

## Typography Presets

The `font_preset` property configures the typography using one of three pre-built configurations [2]:

1. **`"academic"` (Default)**: Optimized for academic research and long-form reading. It uses the classic **Source Serif 4** font for body text and the clean geometric **Inter** font for user interface elements and controls.
2. **`"modern"`**: A clean, highly professional, and geometric appearance. It uses the **Inter** font family for all text elements.
3. **`"humanist"`**: A friendly, warm, and highly accessible aesthetic. It uses the rounded **Nunito** font family for all text elements.

---

## Theme Presets

Siamang provides several pre-configured themes via the `THEME_PRESETS` dictionary. Researchers can retrieve a fully configured `UIConfig` instance using the `get_preset(name)` helper function.

```python
from siamang.frontend import THEME_PRESETS, get_preset

# Retrieve a preset
dark_theme = get_preset("dark")
```

The following theme presets are available:

| Preset Name | Typography | Style / Colors | Best Used For |
| :--- | :--- | :--- | :--- |
| `"default"` | Academic | Light grey background, classic academic styling. | Standard academic and sociological studies. |
| `"academic"` | Academic | Explicit academic styling (alias of `"default"`). | Standard academic and sociological studies. |
| `"dark"` | Academic | Dark slate background (`"#10131a"`), light grey text, and blue accents. | Night-time reading comfort or tech-focused studies. |
| `"modern"` | Modern | Clean white background, crisp borders, indigo accents, and spacious padding. | Public-facing surveys, consumer research, or modern UX testing. |
| `"humanist"` | Humanist | Warm off-white background, soft green accents, rounded corners, and spacious padding. | Community surveys, non-profit research, or friendly feedback forms. |
| `"high_contrast"` | Modern | Black and white palette, bold borders, 18px text, and minimal border-radius. | Accessible surveys adhering strictly to WCAG AAA contrast standards [5]. |

## Runtimes

Runtimes are adapters that convert a compiled survey schema into physical HTML pages and coordinate frontend behavior.

### `RuntimeAdapter` (Abstract Base Class)

All runtimes inherit from this base class, which defines the interface for generating the web survey.

```python
class RuntimeAdapter:
    name: str

    def render_html(self, context: RuntimeRenderContext) -> str: ...
    def render_closed_page(self, context: RuntimeRenderContext, reason: str) -> str: ...
    def stylesheet(self, context: RuntimeRenderContext) -> str | None: ...
    def static_assets(self) -> dict[str, str | bytes]: ...
```

---

### `ReactRuntime`

The `ReactRuntime` is the default and recommended runtime for Siamang surveys. It compiles the questionnaire into a self-contained React 18 single-page application [2].

```python
ReactRuntime(precompile: bool = True)
```

#### Key Features
* **No CDN Dependency**: The runtime ships a pre-compiled, optimized React bundle (`dist/bundle.js`) directly within the survey bundle. It runs entirely offline or from static hosts without loading external scripts (except Google Fonts, if enabled).
* **Native Visibility Engine**: Compiles all `show_if` and `hide_if` expressions into native JavaScript functions, eliminating the overhead of parsing ASTs in the browser [2].
* **Local State Persistence**: Automatically persists current answers to the browser's `localStorage`, allowing respondents to resume their progress if they close the browser tab.

---

### `SurveyJSRuntime`

The `SurveyJSRuntime` is a legacy runtime that renders the survey using the external SurveyJS library. It loads the SurveyJS library from a CDN at runtime.

```python
SurveyJSRuntime(version: str = "1.9.124")
```
